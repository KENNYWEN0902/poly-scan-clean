最重要的结论

1. `.env` 不需要整体重建 只需要手动检查少量字段。最关键的是 POLY_CLOB_HOST、可选的 POLY_RPC_URL，原来的私钥、签名类型、funder 地址一般不要乱动。
2. 你的策略文件不要动 这次升级不要上传 internal/btc/predictor.go、internal/btc/risk_manager.go、internal/btc/strategy.go。
3. 最小升级至少更新 2 个核心文件 scripts/executor.py 和 ecosystem.config.js。前者负责 V2 交互，后者负责把新环境变量送进 PM2 进程。
4. V2 下单看的是 pUSD 不是只有 USDC.e 就行。真正可下单的是 pUSD collateral。
分类 文件
最小升级必须更新 scripts/executor.py、ecosystem.config.js
建议一起更新 scripts/recover_positions.py、scripts/ctf_redeem.py、scripts/claim_rewards.py、scripts/wrap_pusd.py
需要页面和 API 也显示 pUSD 时再更新 cmd/api/main.go、cmd/api/main_test.go、internal/btc/chainlink_monitor.go、internal/btc/predictor.go、internal/polymarket/api.go、dashboard/src/App.tsx、dashboard/src/api.ts、dashboard/src/pages/PnlHistory.tsx、dashboard/src/pages/Portfolio.tsx、dashboard/src/pages/Positions.tsx、dashboard/src/pages/Settings.tsx
仍然不要覆盖 internal/btc/risk_manager.go、internal/btc/strategy.go、服务器上的 .env、data/、logs/
如果你只改了服务器上的 .env，但没有更新 ecosystem.config.js，那 PM2 进程不一定能读到新的 POLY_CLOB_HOST、POLY_RPC_URL、POLY_BUILDER_CODE。这是最常见的“我明明改了配置却不生效”的原因。

4 个基础词

本地 Mac 就是你眼前这台电脑。你的本地项目目录是 /Users/kennywen/Documents/Playground/polymarketbot。

第 0 步：先确认你现在到底在改哪台机器

你会在两个终端之间来回切换：

  • 终端 A：Mac 本地终端。负责上传文件。
  • 终端 B：服务器终端。负责备份、改 .env、安装依赖、重启服务。

先在 Mac 本地终端登录服务器：

ssh root@你的服务器IP

登录成功后先执行：

第 2 步：检查并修改服务器上的 .env

cd /root/poly-scan
grep -n '^POLY_' .env

你重点看这些变量：

  • POLY_PRIVATE_KEY：一般不改
  • POLY_SIGNATURE_TYPE：一般不改
  • POLY_FUNDER_ADDRESS：一般不改
  • POLY_CLOB_HOST：现在想提前试跑 V2 就加
  • POLY_RPC_URL：可选但推荐
  • POLY_BUILDER_CODE：有才填，没有不要乱填

如果你现在就想试跑 V2，用 nano 打开：

nano .env

至少补上：

POLY_CLOB_HOST=https://clob-v2.polymarket.com

如果你有稳定 RPC，再补：

POLY_RPC_URL=https://polygon-mainnet.g.alchemy.com/v2/你的KEY

如果你看到旧 API 三件套：

POLY_API_KEY=...
POLY_API_SECRET=...
POLY_PASSPHRASE=...

建议改成注释：

# POLY_API_KEY=
# POLY_API_SECRET=
# POLY_PASSPHRASE=
不会用 nano 也没关系：
方向键移动光标，Ctrl + O 保存，回车确认，Ctrl + X 退出。
同一个变量不要写两遍。如果文件里已经有 POLY_CLOB_HOST，就修改那一行，不要再复制一份新的。

第 7 步：先判断你是不是 Safe / Proxy 路线

cd /root/poly-scan
grep -n '^POLY_SIGNATURE_TYPE=' .env
grep -n '^POLY_FUNDER_ADDRESS=' .env

你通常会看到以下之一：

  • POLY_SIGNATURE_TYPE=0：更接近普通 EOA 钱包模式
  • POLY_SIGNATURE_TYPE=2：通常是 Proxy / Safe / funder 模式

第 8 步：准备 pUSD

V2 下单最关键的一点是：抵押物是 pUSD，不是只有 USDC.e 就够。

先看余额：

cd /root/poly-scan
set -a
source .env
set +a

python3 scripts/wrap_pusd.py balances

如果你是普通 EOA

python3 scripts/wrap_pusd.py wrap 100
python3 scripts/wrap_pusd.py balances

如果你是 POLY_SIGNATURE_TYPE=2

更稳的做法不是在服务器上直接执行 wrap_pusd.py，而是用真正出资的钱包登录 Polymarket 前端，在前端把资金准备成 pUSD，然后确认 POLY_FUNDER_ADDRESS 对应的钱包里已经有 pUSD。

第 9 步：先做一次最关键的连通性测试

cd /root/poly-scan
set -a
source .env
set +a

echo '{"action":"check_collateral_balance"}' | python3 scripts/executor.py

理想结果类似：

{"success": true, "balance": 123.45}

常见报错含义：

  • ModuleNotFoundError: py_clob_client_v2：新 SDK 没装好
  • Failed to create or derive API key：私钥 / 签名类型 / funder 地址 / 旧 API 三件套有问题
  • insufficient collateral：已经连上了，但 pUSD 不够
  • Failed to get pUSD balance：RPC 或资金状态有问题
如果这一步失败，不要直接重启机器人。先把连通性问题解决掉。

情况 A：只更新 Python 执行层

如果你只更新了 scripts/*.py 和 ecosystem.config.js，那么 poly-bot 二进制一般可以先不重编。

情况 B：还更新了 cmd/api/main.go

cd /root/poly-scan

cd /root/poly-scan/dashboard

第 11 步：如何正确重启 PM2

这次升级有新环境变量，所以最好让 PM2 重新从新版 ecosystem.config.js 加载。

cd /root/poly-scan
pm2 restart ecosystem.config.js --only poly-bot-btc --update-env

如果你还升级了 API / Dashboard，再分别执行：

pm2 restart ecosystem.config.js --only poly-bot-api --update-env
pm2 restart ecosystem.config.js --only poly-dashboard --update-env
pm2 save

第 13 步：如果你现在就想跑 V2

现在就可以提前跑 V2，但你要满足这 4 条：

  1. .env 里有 POLY_CLOB_HOST=https://clob-v2.polymarket.com
  2. scripts/executor.py 已经是新版
  3. py-clob-client-v2 已装好
  4. 你的真实出资地址里已经有 pUSD
提前切到 V2 可以避免以后再做一次代码迁移，但不代表 2026-04-28 19:00 北京时间当天就完全不用管了。那天你仍然要关注官方停机窗口、挂单清空，以及 POLY_CLOB_HOST 从 clob-v2 切回正式入口。

第 14 步：到 2026-04-28 19:00 北京时间当天，你要做什么

如果你在切换前已经写了：

POLY_CLOB_HOST=https://clob-v2.polymarket.com

那么当天按这个顺序做：

  1. 先停机器人：pm2 stop poly-bot-btc
  2. 等官方停机窗口结束
  3. 打开 .env：nano /root/poly-scan/.env
  4. 把 POLY_CLOB_HOST=https://clob-v2.polymarket.com 删除，或者改成 https://clob.polymarket.com
  5. 保存退出：Ctrl + O、回车、Ctrl + X
  6. 重启：pm2 restart ecosystem.config.js --only poly-bot-btc --update-env

如果你还有 API / Dashboard，再分别重启对应进程。

如果升级失败，怎么快速回滚

先停进程：

pm2 stop poly-bot-btc
pm2 stop poly-bot-api
pm2 stop poly-dashboard

回到项目上级目录：

cd /root

把当前目录先改名：

mv poly-scan poly-scan.failed-$(date +%F-%H%M%S)

把备份恢复回来。注意把时间戳替换成你自己的备份名：

cp -a poly-scan.backup-2026-04-22-120000 poly-scan
cp /root/poly-scan/.env.backup-2026-04-22-120000 /root/poly-scan/.env

最后重新启动原服务：

cd /root/poly-scan
pm2 restart ecosystem.config.js --only poly-bot-btc --update-env
pm2 restart ecosystem.config.js --only poly-bot-api --update-env
pm2 restart ecosystem.config.js --only poly-dashboard --update-env
pm2 save

最常见的坑

  1. 把服务器上的 .env 覆盖掉了。
  2. 把自己的策略文件也传上去了。
  3. .env 里写了两行 POLY_CLOB_HOST。
  4. 改了 .env，却没更新 ecosystem.config.js。
  5. 改完后没有用 pm2 restart ... --update-env。
  6. 只有 USDC.e，却以为已经能在 V2 下单。
  7. 在 POLY_SIGNATURE_TYPE=2 模式下直接乱跑 wrap_pusd.py。
  8. 还没做 check_collateral_balance 就直接重启机器人。
  9. 服务器 Python 版本太低。
  10. 切换当天忘了官方会清空挂单。

最稳的执行顺序

  1. 确认服务器真实路径。
  2. 备份整个项目和 .env。
  3. 只手动补 POLY_CLOB_HOST，可选补 POLY_RPC_URL。
  4. 不要改私钥、签名类型、funder 地址。
  5. 从 Mac 只上传最小升级文件。
  6. 不要上传 internal/btc/*。
  7. 安装 py-clob-client-v2。
  8. 准备好 pUSD。
  9. 先做 check_collateral_balance。
  10. 成功后再重启 poly-bot-btc。
  11. 看日志确认没有报错。
  12. 到 2026-04-28 19:00 北京时间，再把 POLY_CLOB_HOST 切回正式入口。
